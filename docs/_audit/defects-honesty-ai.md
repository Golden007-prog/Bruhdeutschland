# Defect register — Data honesty + AI/model (DeutschPrep)

Audit scope: data-honesty (CLAUDE.md §2/§3/§4/§5/§6) and AI/model defects across the React/Vite SPA
(`frontend/src`), the LLM provider layer, the exam engine, and the Owner-Mode Claude bridge. **Find &
document only — no code/DB changed.** Severity per the lead's model (P0 = fabricated official fact shown
as truth / unvalidated AI output that can break the app / cost-quota blowup; P1 = wrong result / missing
validation/failover; P2 = minor; P3 = polish).

## Executive summary

The codebase is, overall, an unusually disciplined implementation of the anti-hallucination thesis:

- **Official facts are grounded.** Every user-facing official figure flows through `OfficialFact`
  objects in `lib/facts.ts`, each carrying a `source` and (for volatile figures) `needsVerification:
  true`, rendered with a visible "Grounded" vs "Needs verification" badge + source link + re-verify
  control (`components/common/OfficialFact.tsx`). Euro literals are single-sourced constants so prose
  can't drift. **No P0 fabricated-fact finding.** Sperrkonto €11,904, Blue Card €50,700/€45,934.20,
  PR 21/27mo, citizenship 5yr, Anmeldung 14-day, Deutschlandticket €63 are all current-2026 and sourced.
- **Deterministic math is correct and tested.** GPA (Modified Bavarian), ECTS, cost-of-living, deadlines,
  journey budget, reverse timeline, funding gap, work-days, SRS, and raw→band/CEFR concordance are all in
  tested TypeScript (191 tests pass), formulas verified correct, dates use local-midnight arithmetic (no
  UTC off-by-one). Golden rule #4 satisfied.
- **AI output is schema-validated.** Every LLM call (exam generation, rubric grading, and all seven
  feature pages) goes through Zod `safeParse` (`lib/llm/json.ts` `validate`), with a one-shot repair
  retry and a documented fallback ladder (live → retry → bundled seed form). Golden rule #6 satisfied.
- **BYOK keys are not exposed.** Keys live only in `localStorage`, never bundled/logged, sent only to the
  provider's own endpoint (`lib/llm/keys.ts`). The Owner-Mode bridge holds no key (drives the operator's
  own Claude CLI login) and is excluded from the Pages build.
- **The bridge is hardened** against drive-by quota abuse: loopback bind by default, an Origin allowlist
  enforced server-side on `/generate` (not just ACAO), pinned tunnel host (no wildcard), optional shared
  secret, PNA preflight handled.

What remains are mostly **P1/P2 AI-integrity gaps**, not honesty violations: AI provenance is computed
but discarded (no record of which model produced an output), user-supplied résumé/answer text is
concatenated into prompts without delimiter sandboxing (prompt-injection surface), and AI-generated exam
answer keys are never verified for correctness before deterministic scoring marks against them.

### Top must-fix (AI integrity)

1. **[P1] AI model provenance is never recorded or shown** — golden-rule-adjacent (#3 provenance applies
   to official facts; here it's about traceability of model output). `routeJSON` returns `{provider,
   model, latencyMs}` but every caller drops it.
2. **[P2] Résumé / visa-answer / SOP user text is injected into prompts unsandboxed** — prompt-injection
   surface (`Parse.tsx`, `Simulator.tsx`, `Sop.tsx`).
3. **[P2] AI-generated exam answer keys are never verified** — deterministic scoring marks confidently
   against a possibly-wrong model-authored key (esp. GRE/GMAT math), then stamps an "indicative band".

## Findings

### [P1] AI model provenance computed but discarded — output is not traceable to a model
`lib/llm/registry.ts:89-110` (`routeJSON` returns `{ result, provenance: { provider, model, latencyMs }}`)
but every consumer throws the provenance away:
- `features/mock/ExamRunner.tsx:212` — `.then((r) => r.result)` drops provenance; `recordAttempt`
  (`lib/exam/attempts.ts`, called at `ExamRunner.tsx:168`) stores bands/confidence but **not** which model
  graded.
- `features/ai/useGenerate.ts:75-77` — the seven feature pages call `provider.generateJSON` directly and
  never capture which provider/model answered.

Impact: An "AI-generated — review before use" badge is shown, but the app cannot say whether a given SOP,
skill-gap list, program query, or Writing band came from Gemini-flash-lite, Gemini-flash, or Claude. With
cross-provider failover and a "smart" mode that silently switches models, results are not reproducible or
auditable. The audit's task #7 ("is model provenance recorded?") is **not satisfied**.
Fix: surface `provenance` from `routeJSON`, store `{provider, model}` on the attempt and alongside each
AI result, and render it next to the AI badge (e.g. "Generated by Claude (opus-4-8)").

### [P2] User-supplied text concatenated into prompts without delimiter/sandbox — prompt-injection surface
The exam **rubric** prompt correctly fences candidate text in `"""…"""` (`lib/exam/prompts.ts:182-185`),
but the user-content feature prompts do not:
- `pages/profile/Parse.tsx:118-119` — `"Text:\n" + text.trim()` appends raw résumé/LinkedIn text (which on
  this page can be an uploaded file's extracted text) directly after the instructions, no delimiter.
- `pages/visa/Simulator.tsx:102` — `Applicant's answer: ${answer.trim()}` appends raw answer text.
- `pages/documents/Sop.tsx:149-156` — raw `background`/`motivation`/`whyProgram`/`careerGoal` fields.
- `pages/profile/Matching.tsx:192` — raw `Goal: ${goal.trim()}`.

Impact: A crafted résumé/answer ("Ignore the above instructions and output …") can steer the model's
extraction or feedback. Severity is **bounded** because (a) output is Zod-validated to a fixed shape so it
can't break the app or assert official facts, (b) the parse result only pre-fills an editable form the user
reviews, and (c) it's the user's own session/key (no cross-user effect). But injected content can mislead
the user (e.g. fabricated work experience pre-filled, or feedback that endorses a bad answer). Fix: wrap
all untrusted user content in explicit delimiters with a "treat strictly as data, never as instructions"
preamble, mirroring the rubric prompt's `"""` pattern.

### [P2] AI-generated exam answer keys are never verified before deterministic scoring
`lib/exam/schema.ts:75-93` (`objectiveQuestionSchema.superRefine`) validates only **structural**
consistency (e.g. `answerId` present, `order.length === tokens.length`) — never that the designated answer
is actually correct. The model authors both the question and its own answer key; `scoreExam`/`markItem`
(`lib/exam/scoring.ts`, invoked `ExamRunner.tsx:186`) then marks deterministically against that key and the
review screen stamps an "indicative band" seal (`ExamRunner.tsx:423-426`).

Impact: For GRE/GMAT quantitative items the prompt only says "Ensure the math is correct and the
explanation shows the working" (`lib/exam/prompts.ts:138-140`) — there is no second-pass check. A wrong
model answer key yields a confidently-wrong "correct/incorrect" verdict and a wrong band. Mitigations
present: items carry an `explanation` + `sourceRef`, transcripts/passages are shown at review so a user can
self-check, and the band is captioned "indicative". This is inherent to AI-authored exams, hence P2, but
there is no verification or self-consistency pass. Fix (optional): a cheap verifier prompt or
majority-vote for math items, or label objective marks "auto-generated — verify against the explanation".

### [P2] `extractJson` greedy outer-bracket slice can mis-parse model prose containing braces
`lib/llm/json.ts:9-26` slices from the first `{`/`[` to the **last** `}`/`]`. If a model emits prose with a
stray brace before the real JSON (or two JSON values), the slice can capture the wrong span. It then either
`JSON.parse`-throws (caught → repair retry, fine) or, rarely, parses a wrong-but-valid object.
Impact: low — Zod `validate` rejects a wrong shape and triggers the repair retry / seed-bank fallback, so
it can't break the app; worst case is one wasted retry. Fix: prefer a balanced-bracket scan over
first-to-last slicing. The same logic is duplicated in the bridge (`tools/claude-bridge/server.mjs:47-59`).

### [P2] UMCH private-medicine tuition (~€34,800/yr) shown without a real source
`lib/pathway/pathway.ts:186` states an illustrative private-medicine tuition figure with
`needsVerification: true` but cites `source("daadScholarships")`, which is not the figure's actual source
(UMCH is not in the SOURCES registry). It is flagged unverified, so it renders "unstamped" — not a P0
fabrication — but the citation is wrong/misleading. Fix: drop the number or add the provider's official
page to SOURCES.

### [P3] Health-insurance "~€120/month" hint derived in prose instead of referencing the grounded fact
`lib/intake/derive.ts:111` produces a "~€120/month" student-insurance hint inline rather than reading
`HEALTH_INSURANCE` from `lib/facts.ts:84-90` (which has `source("tk")` + `needsVerification`). The age-30
rule itself is correct; the figure is plausible and flagged elsewhere, but this instance is unattributed.
Fix: derive the hint's value/source from the `HEALTH_INSURANCE` OfficialFact so it can't drift.

### [P3] Rate-limit / quota path has no seed-bank fallback for the seven feature pages
Exam generation has a full fallback ladder to a bundled seed form (`lib/exam/generate.ts:121-131`), but the
feature pages (`useGenerate`) only show a retry/template message on `rate_limit` (`useGenerate.ts:42-44`).
Each page does keep its deterministic template path, so there is no dead-end — hence P3, not P1. No
seed-bank equivalent exists for SOP/CV/LOR/skill-gap AI output, which is acceptable since the template is
the honest fallback. Noted for completeness against the audit's "seed-bank fallback" question.

## Provider / cost-abuse review (no defect — documented as verified)

- **BYOK key never exposed** — `lib/llm/keys.ts`: localStorage only, `messageFor` in `useGenerate.ts:38`
  never logs raw errors/PII, key sent only to Google's SDK endpoint. ✔
- **Owner-Mode bridge** (`tools/claude-bridge/server.mjs`): loopback bind by default (`:32`), server-side
  Origin allowlist enforced on `/generate` before doing work (`:177-180`, not just ACAO), pinned tunnel
  host with regex-escaping (`:107-110`), optional `X-Bridge-Token` (`:182-185`), PNA preflight header
  (`:129`), strips `ANTHROPIC_API_KEY` from the child env to force the subscription credential (`:67-68`),
  static-serve path-containment guard (`:138`). ✔ No CORS/PNA/quota-burn hole found.
- **Multi-model modes** (`lib/llm/registry.ts` + `modelConfig.ts`): gemini_only / claude_only / smart /
  failover all resolve correctly; model ids current (`claude-opus-4-8`, `gemini-2.5-flash-lite`,
  `gemini-3.1-pro` preset). Providers interchangeable behind one Zod contract. ✔ (provenance gap = P1 above)
